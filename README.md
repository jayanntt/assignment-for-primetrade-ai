# PrimeTrade AI Assignment

This repository contains a data analysis project for linking Fear/Greed sentiment data with trading history. The goal is to merge market sentiment with trade outcomes and generate exploratory analytics and strategy guidance.

## Repository Contents

- `work.py` - Main analysis script that loads sentiment and trade data, normalizes timestamps, merges datasets, and creates visualizations.
- `fear_greed_index.csv` - Daily market sentiment dataset with `date`, `value`, and `classification` fields.
- `historical_data.csv` - Trade history dataset containing trade metadata such as account, execution price, side, timestamp, and closed PnL.
- `inspect_dates.ps1` - PowerShell helper that inspects date alignment between the sentiment and trade datasets.
- `plot_fear_vs_greed_pnl.png` - Example output generated by `work.py`.
- `plot_win_rate_by_side_and_sentiment.png` - Example output generated by `work.py`.

## What the script does

`work.py` performs the following tasks:

1. Loads and normalizes the Fear/Greed sentiment dataset.
2. Loads and standardizes the trading dataset.
3. Parses and normalizes trade datetime values.
4. Merges trade records with sentiment labels by date.
5. Generates exploratory visualizations:
   - Average closed PnL for Fear vs Greed regimes
   - Average leverage by sentiment classification
   - Win rate by trade direction and sentiment group
6. Prints summary statistics and analysis to the console.

## Requirements

- Python 3.8+
- `pandas`
- `numpy`
- `matplotlib`

Install dependencies with:

```powershell
python -m pip install pandas numpy matplotlib
```

## Usage

Run the analysis from the repository root:

```powershell
python work.py
```

The script will generate PNG files for the charts and print analysis summaries to the terminal.

## Notes

- The script expects the two CSV files to be in the same directory as `work.py`.
- Trade timestamps are parsed from `Timestamp IST` with the format `dd-MM-yyyy HH:mm`. If that field is not available, the script will fallback to an epoch-style `Timestamp` column when present.
- Sentiment classifications are grouped into `Fear`, `Greed`, and `Neutral` for aggregate analysis.
